## 🏗️ 迭代6-8架构更新

### 核心架构变更

**采用双层抽象架构**：
```
领域层：Actor值对象（轻量级抽象）
    ↕ 仓储层自动转换
数据库层：Participant实体（外键保证）
```

**Agent作为一等公民**：
- Agent使用API Key独立认证
- Agent可以管理contacts、创建会话、发送消息
- 与User在API层面完全平等

## 🔄 迭代原则

### 每个迭代的标准流程

1. **计划阶段** (10%)
   - 明确迭代目标
   - 列出具体任务
   - 确定验收标准

2. **开发阶段** (70%)
   - 按照编码规范开发
   - 每次只做一个功能点
   - 及时提交代码

3. **测试阶段** (15%)
   - 编写单元测试
   - 手动测试验证
   - 修复发现的问题

4. **总结阶段** (5%)
   - 更新文档
   - 记录经验教训
   - 准备下一个迭代

### 迭代完成标准

- ✅ 所有计划功能已实现
- ✅ 代码通过测试
- ✅ 文档已更新
- ✅ 可以独立演示
